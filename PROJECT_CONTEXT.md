\# PROJECT\_CONTEXT.md



\## 1. 作业背景



这是 Java 程序设计课程期末大作业。老师提供了多个选题，本项目最终选择：



一带一路沿线国家合作态势分析系统



项目需要基于 GDELT 2.0 Event 数据，开发一个 Java 桌面窗口程序，用于分析一带一路沿线国家之间的合作态势、风险变化和区域差异。



本项目只做这个方向，不做其他选题。



\## 2. 老师的总要求



根据老师提供的要求，本项目需要注意：



1\. 大作业是 Java 课程期末大作业。

2\. 程序应为窗口程序。

3\. 涉及固定文件存放时，统一放在 D 盘 Temp 下。

4\. 最终需要提交源代码、可执行程序、示例数据、设计文档、实验报告、答辩 PPT 等材料。

5\. 作业和 PPT 上需要写明组员姓名、学号和分工。

6\. 汇报时间有限，系统需要适合现场演示，不能只停留在代码层面。



\## 3. 评分导向



老师评分不仅看功能，还看以下方面：



1\. 程序设计思路是否完整。

2\. 是否能清楚说明程序功能、结构和流程。

3\. 是否结合 Java 课程理论知识。

4\. 项目是否有一定难度和工作量。

5\. 程序是否能正确运行。

6\. 代码是否规范，命名、方法设计和注释是否清楚。

7\. 界面是否合理、友好、完整。

8\. 答辩展示和报告质量是否充分。



因此，本项目不能只追求堆功能，更要保证：



\* 架构清晰

\* 代码规范

\* 可运行

\* 可展示

\* 可解释

\* 适合写报告和答辩



\## 4. 选题要求概括



本项目围绕 GDELT 2.0 Event 数据和一带一路沿线国家展开。



老师任务书中希望系统能够完成：



1\. 数据导入与管理

2\. 数据清洗

3\. 数据库存储

4\. 国家配置与区域分类

5\. 多条件查询

6\. 中国与沿线国家双边关系查询

7\. 合作事件和冲突事件筛选

8\. 合作指数计算

9\. 趋势分析

10\. 区域聚类分析

11\. 投资或合作风险评估

12\. 首页仪表盘

13\. 图表可视化

14\. 专题地图或区域展示

15\. 分析结果导出



这些功能可以根据实现难度进行合理取舍，但最终系统应形成完整闭环：



数据导入 → 数据存储 → 查询检索 → 指标分析 → 风险评估 → 图表展示 → 结果导出



\## 5. 数据说明



GDELT 数据通常是无表头文本数据，字段较多。本项目不需要处理所有字段，但应重点关注：



\* 事件编号

\* 事件日期

\* Actor1 国家代码

\* Actor2 国家代码

\* 事件编码

\* 事件根编码

\* GoldsteinScale

\* NumMentions

\* AvgTone

\* 经纬度信息



系统应能根据事件编码或事件根编码区分合作、冲突和其他事件。



\## 6. 项目开发目标



本项目希望做成一个完整、稳定、适合答辩展示的 Java 桌面应用，而不是简单脚本。



请 Codex 在设计和开发时主动思考：



1\. 哪些功能是高分必须做的。

2\. 哪些功能可以简化但保留展示效果。

3\. 哪些功能适合作为亮点或加分项。

4\. 如何设计更清晰的软件架构。

5\. 如何让系统稳定运行。

6\. 如何让界面适合课堂展示。

7\. 如何让代码方便写报告解释。

8\. 如何通过 GitHub 提交记录体现工作量。



\## 7. 开发环境倾向



当前计划使用：



\* Codex 软件辅助开发

\* VS Code 查看、运行和管理项目

\* GitHub 保存代码和版本记录

\* Java 17

\* Maven



具体采用 Swing、JavaFX、SQLite、Access、JFreeChart 或其他技术方案，可以由 Codex 根据稳定性、实现难度和课程适配性进行判断，但必须说明理由。



\## 8. 开发原则



请不要一开始直接生成完整大系统。



请先进行项目分析和方案设计，再分阶段开发。每个阶段都应该能形成一个可以运行、可以提交 GitHub 的版本。



优先级为：



1\. 程序能运行

2\. 结构清晰

3\. 功能闭环完整

4\. 代码规范

5\. 界面友好

6\. 分析模型有解释性

7\. 答辩展示效果好
