# 一带一路沿线国家合作态势分析系统实验报告

## 1. 实验目的

通过 Java 桌面应用开发，完成一个基于 GDELT 2.0 Event 数据的一带一路沿线国家合作态势分析系统。实验目标包括掌握 JavaFX 界面开发、SQLite 数据库访问、文件解析、分层架构设计和数据可视化。

## 2. 开发环境

- 操作系统：Windows
- JDK：Java 17
- 构建工具：Maven
- 界面技术：JavaFX
- 数据库：SQLite
- 数据来源：GDELT 2.0 Event
- 运行期目录：`D:\Temp\BRI-GDELT-System`

## 3. 功能实现

系统已实现以下功能：

- 数据导入：支持 GDELT `.CSV/.tsv/.zip`，按一带一路国家过滤，批量写入 SQLite。
- 首页仪表盘：展示事件总量、合作/冲突结构、国家事件量和月度趋势。
- 事件查询：支持日期、国家代码和事件类型检索。
- 双边关系：支持中国与沿线国家的双边统计、月度趋势和事件明细。
- 合作态势分析：根据合作事件量、Goldstein、AvgTone、媒体关注度计算合作指数。
- 风险评估：根据冲突事件、负向指标和冲突占比计算风险指数。
- 专题地图：使用 ActionGeo 经纬度展示事件空间分布。
- 结果导出：生成 TXT 汇总报告、合作排名 CSV、风险排名 CSV。

## 4. 关键实现过程

系统首先建立统一的数据目录和 SQLite 表结构，然后通过 `GdeltImportService` 读取原始 GDELT 文件。解析时使用固定字段映射，遇到字段不足、数字异常、经纬度异常等情况会跳过或置空，避免程序崩溃。

入库后，系统通过 Repository 层提供查询和聚合接口，Service 层负责业务校验和指标计算，JavaFX UI 层负责展示。结果导出服务将当前数据库中的统计和排名结果写为报告文件。

## 7. 问题与解决方案

- GDELT 文件无表头且字段多：通过固定字段映射类统一维护字段下标。
- 原始数据可能存在坏行：解析器采用安全解析，异常行跳过并记录错误样例。
- 重复导入风险：事件表使用 `global_event_id` 主键并采用 `INSERT OR IGNORE`。
- 大量数据影响界面响应：导入和查询使用 JavaFX 后台任务，避免界面卡顿。
- PPT 需要清晰结果：增加指标卡片、研判文本、图表和导出文件。

## 8. 总结
