# 一带一路沿线国家合作态势分析系统设计文档

## 1. 项目背景

本项目是 Java 程序设计课程期末大作业，选题为“一带一路沿线国家合作态势分析系统”。系统基于 GDELT 2.0 Event 数据，围绕沿线国家事件、合作关系和风险变化进行导入、查询、统计、可视化与结果导出。

## 2. 需求分析

系统围绕“数据导入 -> 数据存储 -> 查询检索 -> 指标分析 -> 风险评估 -> 可视化 -> 结果导出”的闭环设计，核心需求包括：

- 支持 GDELT 2.0 `.CSV/.tsv/.zip` 文件导入。
- 根据一带一路国家配置过滤事件。
- 保存事件编号、日期、国家代码、事件编码、GoldsteinScale、NumMentions、AvgTone 和经纬度。
- 支持按日期、国家和事件类型查询事件。
- 支持中国与沿线国家双边关系统计。
- 支持合作指数、风险指数、月度趋势和专题地图展示。
- 支持导出汇总报告和排名 CSV。

## 3. 技术路线

- Java 17：满足课程 Java 开发要求。
- JavaFX：实现桌面窗口程序和图表界面。
- SQLite：轻量本地数据库，便于提交和离线运行。
- Maven：统一依赖管理、编译和运行。
- GDELT 2.0 Event：作为国际事件数据来源。

## 4. 软件架构

系统采用分层结构：

- UI 层：`MainView` 构建 JavaFX 页面，包括仪表盘、导入、查询、双边关系、合作态势、风险评估、专题地图和结果导出。
- Service 层：封装业务逻辑，如导入、查询、双边分析、仪表盘统计、合作/风险分析、地图点位和结果导出。
- Repository 层：封装 SQLite 访问，负责国家、事件和导入批次的增删查统计。
- Domain 层：定义业务模型，如 `GdeltEvent`、`DashboardSummary`、`BilateralRelationSummary`、`CooperationScore`、`RiskAssessment`。
- Util 层：处理 GDELT 字段映射和安全解析。

## 5. 数据流

1. 用户或脚本将 GDELT 原始文件放入 `D:\Temp\BRI-GDELT-System\input`。
2. 导入服务读取普通文本或 zip 文件，按 Tab 分隔解析字段。
3. 系统根据 `countries` 表中的 CAMEO 国家代码过滤一带一路相关事件。
4. 清洗后的事件批量写入 `gdelt_events`，导入批次写入 `import_batches`。
5. 查询和分析页面通过 Service 层读取 Repository 聚合结果。
6. JavaFX 页面展示图表、表格、指标卡片和研判文本。
7. 结果导出服务生成 TXT 汇总报告、合作排名 CSV 和风险排名 CSV。

## 6. 数据库设计

主要表结构：

- `countries`：保存国家代码、中文名、英文名、区域、经纬度、是否一带一路国家等信息。
- `gdelt_events`：保存 GDELT 事件主体字段，`global_event_id` 作为主键，重复导入时使用 `INSERT OR IGNORE` 避免重复。
- `import_batches`：记录导入文件名、时间、总行数、成功行数、跳过行数、状态和错误摘要。
- `analysis_results`：分析结果扩展表，当前核心统计主要通过 SQL 聚合实时生成。

关键索引包括事件日期、Actor1、Actor2、事件类型、事件根编码和双边关系日期索引，以支持查询和聚合。

## 7. 核心算法与指标

事件分类：

- `EventRootCode` 为 `04/05/06` 时归为合作事件。
- `EventRootCode` 为 `08-14` 时归为冲突事件。
- 其他事件归为 `OTHER`。

合作指数综合考虑：

- 合作事件量
- 正向 GoldsteinScale
- 正向 AvgTone
- NumMentions 媒体关注度
- 冲突事件扣分

风险指数综合考虑：

- 冲突事件量
- 冲突占比
- 负向 GoldsteinScale
- 负向 AvgTone

风险等级划分为低、中、高、极高四档，便于 PPT 中解释。

## 8. 界面设计

系统界面采用蓝白灰配色，强调课程项目的清晰、稳重和可读性。首页展示总体数据，分析页使用指标卡片、图表、表格和简短研判说明，使 PPT 截图可以直接表达系统功能和分析结果。

## 9. 项目亮点

- 完成了从真实 GDELT 文件到 SQLite 入库、查询、分析、地图、导出的完整闭环。
- 使用分层架构，便于说明 Java 面向对象设计和模块职责。
- 指标模型具有可解释性，适合答辩说明。
- 支持真实数据批量导入和重复导入去重。
- 输出报告和 CSV 文件，便于整理实验报告和 PPT。
