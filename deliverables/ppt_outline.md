# 一带一路沿线国家合作态势分析系统答辩 PPT 详细设计方案

> 当前文件是 PPT 制作方案，先供审核。正式 PPT 建议控制在 12-14 页，符合老师要求的 10-15 页范围。整体风格建议使用深蓝、青绿、白色和浅灰，少用大段文字，多放系统截图、流程图、指标公式和关键数据。

## 1. 总体策略

答辩时间通常不超过 10 分钟，PPT 不宜做成说明书。建议围绕“为什么做、怎么做、做成什么、结果可靠吗、有什么亮点”展开。

本项目的优势是：系统闭环完整、真实数据规模较大、分层结构清楚、合作/风险指标可解释、导出链路完整。需要避开的短板是：聚类模型是轻量实现，不要夸大为复杂机器学习平台；国家配置当前为 145 个，不要写死为已完整覆盖 150+；地图是事件点位和低分辨率底图，不要说成专业 GIS 平台。

## 3. 逐页设计

### 第 1 页：封面

标题：一带一路沿线国家合作态势分析系统

副标题：基于 GDELT 2.0 Event 数据的 Java 桌面分析系统

页面元素：

1. 项目图标或系统首页局部截图。
2. 课程名称：Java 程序设计。
3. 组员姓名、学号、分工。
4. 日期：2026 年 6 月。

讲述重点：

本项目使用 JavaFX、SQLite 和 GDELT 事件数据，实现沿线国家合作、风险和区域差异分析。

### 第 2 页：选题背景

标题：为什么选择这个题目

内容建议：

1. 一带一路沿线国家互动频繁，合作机会和风险信号并存。
2. GDELT 记录全球事件，包含国家、事件编码、Goldstein、媒体语调和地理信息。
3. 本项目用 Java 桌面程序把原始事件数据转化为可查询、可分析、可展示的结果。

视觉建议：

左侧放一带一路/世界地图背景，右侧用 3 个短句概括。不建议放长段背景介绍。

### 第 3 页：任务要求与需求拆解

标题：从任务书拆解出的核心需求

内容建议使用 2 行 4 列矩阵：

1. 数据导入与清洗。
2. SQLite 数据库存储。
3. 多条件事件查询。
4. 中国与沿线国家双边关系。
5. 合作指数与趋势。
6. 风险评估。
7. 区域分析与国家聚类。
8. 专题地图与结果导出。

可加一句：系统实现目标不是只做单个图表，而是形成“导入 -> 入库 -> 查询 -> 分析 -> 可视化 -> 导出”的闭环。

### 第 4 页：系统总体架构

标题：分层架构设计

图示建议：

```text
JavaFX UI
  ↓
Service 业务层
  ↓
Repository 数据访问层
  ↓
SQLite 数据库

Domain 模型 + Util 工具类贯穿支撑
```

每层说明：

1. UI：窗口、导航、图表、表格、后台任务。
2. Service：导入、查询、双边、分析、地图、导出。
3. Repository：SQL 查询、批量入库、统计聚合。
4. Domain/Util：事件对象、指标对象、字段映射、安全解析。

答辩话术：

“这样划分后，界面不直接写 SQL，数据库层也不处理界面逻辑，代码职责更清楚。”

### 第 5 页：数据流程与数据库设计

标题：从 GDELT 文件到分析结果

数据流程图：

```text
GDELT zip/csv/tsv
 -> Tab 分隔解析
 -> 国家配置过滤
 -> 经纬度和异常值清洗
 -> SQLite 批量入库
 -> 指标聚合
 -> 图表/地图展示
 -> TXT/CSV/XLSX 导出
```

数据库表：

1. `countries`：国家元数据和区域。
2. `gdelt_events`：清洗后的事件主表。
3. `import_batches`：导入批次。
4. `analysis_results`：分析结果扩展。

强调点：

`global_event_id` 是主键，重复导入不会重复计数；事件表建立日期、国家、类型和双边关系索引。

### 第 6 页：数据导入与清洗

标题：原始数据处理的关键点

内容建议：

1. GDELT 文件无表头，系统用固定字段映射读取关键字段。
2. 支持 `.CSV`、`.tsv`、`.zip`。
3. 异常行跳过并记录错误样例。
4. 经纬度超出范围时置空。
5. 每 1000 条批量写入 SQLite。

可展示字段表：

| 字段 | 含义 |
|---|---|
| GlobalEventID | 事件唯一编号 |
| SQLDATE | 日期 |
| Actor1/Actor2 | 国家代码 |
| EventRootCode | 合作/冲突分类 |
| GoldsteinScale | 事件强度 |
| NumMentions | 媒体关注度 |
| AvgTone | 媒体语调 |
| ActionGeo | 经纬度 |

### 第 9 页：合作指数模型与结果

标题：合作指数：把事件转化为可比较排名

公式建议：

```text
合作指数 =
合作占比
+ 合作事件相对规模
+ 正向 Goldstein
+ 正向 AvgTone
+ 媒体关注度
- 冲突占比扣分
```

正式口径可简写为：

```text
cooperationIndex = clamp(
  100 * (
    cooperationRatio * 0.35
    + cooperationShare * 0.30
    + positiveGoldstein * 0.15
    + positiveTone * 0.08
    + mentionShare * 0.12
    - conflictRatio * 0.15
  )
)
```

截图建议：

放合作态势页面或合作排名图。

讲述重点：

指标不只看事件数量，还考虑事件强度、媒体语调和媒体关注度；冲突事件会拉低合作指数。

### 第 10 页：风险评估模型与结果

标题：风险指数：识别潜在冲突和负面信号

公式建议：

```text
风险指数 =
冲突占比
+ 冲突事件相对规模
+ 负向 Goldstein
+ 负向 AvgTone
```

风险等级：

| 分数 | 等级 |
|---|---|
| 0-25 | 低 |
| 25-50 | 中 |
| 50-75 | 高 |
| 75-100 | 极高 |

截图建议：

放风险评估页面，包含风险排名、等级和说明。

讲述重点：

风险评估用于辅助判断哪些国家需要重点关注，不直接等同于现实投资结论。
